# 变量相关知识

# 变量提升

JS解析器在执行代码之前，会对代码进行解析，会将 var 声明的变量的声明放到前面先提前声明，这就是变量提升。

var name1 = '小张';
console.log(name1);
1
2

在解析后会变成这样

var name1;
name1 = '小张';
console.log(name1); // undefined
1
2
3

在if语句中也会出现变量提升

if(false){
    var name1 = '小张';
}
console.log(name1); // undefined

# TDZ暂时性死区

TDZ暂时性死区，意思是变量在作用域中已经存在，但是由于变量先声明后使用的问题造成了TDZ，let、const 变量先声明或者先声明并赋值才可以使用。

let name1 = '小张';
function aaa(){
    console.log(name1);
}
aaa(); // 小张
1

let name1 = '小张';
function aaa(){
    console.log(name1);
    let name1 = '小王';
}
aaa(); // 报错

6

函数内部如有name1变量，则就近原则（作用域）使用函数内部的变量，否则使用函数外部变量

建议：变量先声明或者先声明并赋值再使用，少用 var 声明变量，多使用 let、const 声明变量和常量

# var

var 声明变量是没有块作用域的，这会导致全局污染

var a = 10;
for(var a = 0;a<4;a++){
    console.log(a); // 0 1 2 3
}
console.log(a); // 4
1
2
3
4
5

第5行输出a变量就受污染了，影响了全局

使用 let 声明则不会出现该情况

let a = 10;
for(let a = 0;a<4;a++){
    console.log(a); // 0 1 2 3
}
console.log(a); // 10

let 是有块作用域的，这样就不会全局污染

console.log($.name1);

# const

const 用来定义常量，受作用域影响

  • 一般写程序常量都使用大写
  • 不允许修改常量的值（引用类型除外）
  • 在声明常量是必须同时进行赋值
  • 有受块、函数、全局作用域

不允许重新赋值

const NAME = '小张';
NAME = '小汪'; // 报错 Uncaught TypeError: Assignment to constant variable.
1
2

修改引用类型的常量

const INFO = {
    name: '小张',
    age: 28
};
INFO.age = 20;
console.log(INFO.age); // 20
1

在不同的作用域可重新定义

const NAME = '小张';
function aaa(){
    const NAME = '小汪';
    console.log(NAME);
}
{
    const NAME = '小林';
    console.log(NAME); // 小林
}
aaa(); // 小汪
console.log(NAME); // 小张

var name1 = 'aaaaa';

1
2

JS中使用 var 定义的变量可以使用window对象进行调用

var screenTop = 'aaaaa';

1
2

若定义的变量名与window对象的全局变量名相同则会污染

# 规范重复声明

为了避免出现同一个作用域出现重复声明情况，要进行规范化

使用 let const变量避免出现重复声明，提醒我们重复定义了

let name1 = 'aaaa';
let name1 = 'bbbb';
console.log(name1); // 报错 Uncaught SyntaxError: Identifier 'name1' has already been declared
1
2
3
const name1 = 'aaaa';
const name1 = 'bbbb';
console.log(name1); // 报错 Uncaught SyntaxError: Identifier 'name1' has already been declared

# 冻结变量（Object.freeze）

在定义常量时，引用类型常量的值是可以被改变的，若想要不被改变可使用冻结变量 Object.freeze

const INFO = {
    name: '小张',
    age: 20
};
Object.freeze(INFO);
INFO.age = 28;
console.log(INFO.age); // 20

# 变量的传值与传址

传值一般是基本数据类型（标量）如：数值、字符串等简单的数据类型

传址一般是引用数据类型如：对象数据类型等

# 传值

let name1 = '小张';
let name2 = name1;
name2 = '小汪';
console.log(name1); // 小张
console.log(name2); // 小汪
1
2
3
4
5

因为 name1 变量和 name2 变量使用的不是同一个内存地址，而是 name1 变量将值复制一份给了 name2 变量，因此改变 name2 变量的值并不会改变 name1 变量的值

# 传址

let info = {
    name: '小张',
    age: 20
};
let info2 = info;
info2.name = '小汪';
console.log(info.name); // 小汪
console.log(info2.name); // 小汪
1
2
3
4
5
6
7
8

变量 info2 保存的是 info 变量值的内存地址，修改 info2 变量的值等同于修改 info 变量的值。

# 严格模式

严格模式用来规范代码，主流框架都采用了严格模式，建议代码使用严格模式开发

变量必须使用变量声明关键词声明，未声明变量不允许赋值

"use strict";
a = 1;
console.log(a); // 报错 Uncaught ReferenceError: a is not defined
1
2
3

关键词不能作为变量名使用

"use strict";
var public = 'aaa';
console.log(public); // 报错 Uncaught SyntaxError: Unexpected strict mode reserved word
1
2
3

在非严格模式中，声明变量可不用声明指令

// "use strict";
({name,age} = {name:'小张',age:20});
console.log(name); // 小张
console.log(age); // 20

    label2: for (let j = 1; j <= 10; j++) {
        if (j % 2 != 0) {
            continue label2;
        }
        console.log(i, j);
        if (i + j > 10) {
            break label1;
        }
    }
}
