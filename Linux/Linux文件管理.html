# 绝对路径和相对路径

绝对路径：由根目录 / 写起，例如：/usr/share/doc 这个目录，如果切换目录知道绝对路径可直接使用绝对路径切换 cd /usr/share/doc

相对路径：不是由 / 写起，若 /usr/share/doc 要到 /usr/share/man 目录下时，由于 doc 和 man 都是在 /usr/share 目录下，则 doc 和 man 在同一级目录，当前是在 /usr/share/doc，可以先返回上一级到 /usr/share/ 目录下，然后再切换到 man 目录下，则可以写成 cd ../man 这就是相对路径的写法

. 为当前路径

.. 为上一级目录

在切换目录是或者操作文件时，我们则需要加 /，例如 ./ 当前路径 ../ 上一级目录

# 文件管理命令

# ls

ls 显示指定工作目录下的内容及属性信息

ls 命令是Linux下最常用的指令之一。ls命令为英文单词 list 的缩写，正如英文单词 list 的意思，其功能是列出指定目录下的内容及其相关属性信息。

默认状态下，ls命令会列出当前目录的内容。而带上参数后，我们可以用ls做更多的事情。

语法格式：ls [参数] [文件]

常用参数：

参数 描述
-a 显示所有文件及目录（包括以.开头的隐藏文件）
-l 使用长格式列出文件及目录信息（详细信息）
-r 将文件以相反次序显示（默认按照英文字母排序）
-t 根据最后的修改时间排序
-A 同 -a，但是不列出 .（当前目录）和..（父目录）
-S 根据文件大小排序
-R 递归列出所有子目录

[parallels@centos-7 ~]$ ls -l

[parallels@centos-7 ~]$ ls -r

[parallels@centos-7 ~]$ ls -t

[parallels@centos-7 ~]$ ls -A

[parallels@centos-7 ~]$ ls -S

[parallels@centos-7 ~]$ ls -R

# cd

cd命令是 change directory 中单词的首字母缩写，其英文释义是改变目录，所以该命令的功能是从当前目录切换到指定目录。

其中目录的路径可分为绝对路径和相对路径。若目录名称省略，则切换至使用者的用户目录(也就是刚登录时所在的目录)。

另外，~ 也表示为用户目录的意思，. 则是表示目前所在的目录，.. 则表示当前目录位置的上一级目录。

语法格式：cd [目录名]

常用参数：

参数 描述
~ 切换至当前用户目录
.. 切换至当前目录位置的上一级目录

将当前工作目录切换到dir目录，并使用pwd命令查看当前目录

# cp

cp命令可以理解为英文单词copy的缩写，其功能为复制文件或目录。

cp命令可以将多个文件复制到一个具体的文件名或一个已经存在的目录下，也可以同时复制多个文件到一个指定的目录中。

语法格式：cp [参数] [文件]

参数 描述
-f 若目标文件已存在，则会直接覆盖原文件
-i 若目标文件已存在，则会询问是否覆盖（不常用）
-p 保留源文件或目录的所有属性（不常用）
-r 递归复制文件和目录
-d 当复制符号连接时，把目标文件或目录也建立为符号连接，并指向与源文件或目录连接的原始文件或目录（不常用）
-l 对源文件建立硬连接，而非复制文件
-s 对源文件建立符号连接，而非复制文件
-b 覆盖已存在的文件目标前将目标文件备份（不常用）
-v 详细显示cp命令执行的操作过程（不常用）
-a 等价于“dpr”选项（不常用）

复制目录

[root@localhost ~]# cp -R dir1 dir2/
1

将文件test1改名为test2

[root@localhost ~]# cp -f test1 test2
1

复制多个文件到 dir

[root@localhost ~]# cp -r file1 file2 file3 dir
1

交互式地将目录 /root/ 中的所有 .c 文件复制到目录 dir 中

[root@localhost ~]# cp -r /root/*.c dir
1

# mkdir

mkdir命令是 make directories 的缩写，用来创建目录。

注意：默认状态下，如果要创建的目录已经存在，则提示已存在，而不会继续创建目录。 所以在创建目录时，应保证新建的目录与它所在目录下的文件没有重名。

语法格式 : mkdir [参数] [目录]

常用参数：

参数 描述
-p 递归创建多级目录
-m 建立目录的同时设置目录的权限
-v 显示目录的创建过程

在当前工作目录下，建立一个名为 dir 的子目录

[root@localhost ~]# mkdir dir
1

在目录 /root/ 下建立子目录dir，并且设置文件属主有读、写和执行权限，其他人无权访问

[root@localhost ~]# mkdir -m 700 /root/dir
1

同时创建子目录 dir1 dir2 dir3

[root@localhost ~]# mkdir dir1 dir2 dir3
1

递归创建多级目录

[root@localhost ~]# mkdir -p abc/dir
1

# mv

mv命令是 move 单词的缩写，其功能大致和英文含义一样，可以移动文件或对其改名。

这是一个使用频率超高的文件管理命令，我们需要特别留意它与复制的区别：mv与cp的结果不同。mv命令好像文件搬家，文件名称发生改变，但个数并未增加。而cp命令是对文件进行复制操作，文件个数是有增加的。

语法格式：mv [参数]

常用参数：

参数 描述
-i 若存在同名文件，则向用户询问是否覆盖（不常用）
-f 覆盖已有文件时，不进行任何提示
-b 当文件存在时，覆盖前为其创建一个备份（不常用）
-u 当源文件比目标文件新，或者目标文件不存在时，才执行移动此操作（不常用）

将文件 file_1 重命名为 file_2

[root@localhost ~]# mv file_1 file_2
1

将文件 file 移动到目录 dir 中

[root@localhost ~]# mv file /dir
1

将目录 dir1 移动目录 dir2 中（前提是目录dir2已存在，若不存在则改名)

[root@localhost ~]# mv /dir1 /dir2
1

将目录 dir1 下的文件移动到当前目录下

[root@localhost ~]# mv /dir1/* .
1

# pwd

pwd命令是 print working directory 中每个单词的首字母缩写，其功能正如所示单词一样，为打印工作目录，即显示当前工作目录的绝对路径。

在实际工作中，我们经常会在不同目录之间进行切换，为了防止迷路，我们可以使用pwd命令快速查看当前我们所在的目录路径。

[root@localhost ~]# pwd
/root

# cat

Linux系统中有很多个用于查看文件内容的命令，每个命令又都有自己的特点，比如这个cat命令就是用于查看内容较少的纯文本文件的

语法格式：cat [参数] [文件]

常用参数：

参数 描述
-n 显示行数（空行也编号）
-s 显示行数（多个空行算一个编号）

查看文件的内容

查看文件的内容，并显示行数编号

清空文件的内容

[root@localhost ~]# cat /dev/null > /root/filename.txt

持续写入文件内容，碰到EOF符后结束并保存

[root@localhost ~]# cat > filename.txt <<EOF

# rm

rm是常用的命令，该命令的功能为删除一个目录中的一个或多个文件或目录，它也可以将某个目录及其下的所有文件及子目录均删除。对于链接文件，只是删除了链接，原有文件均保持不变。 rm也是一个危险的命令，使用的时候要特别当心，尤其对于新手，否则整个系统就会毁在这个命令（比如在/（根目录）下执行rm * -rf）。所以，我们在执行rm之前最好先确认一下在哪个目录，到底要删除什么东西，操作时保持高度清醒的头脑。

语法格式：rm [参数] [文件]

常用参数：

参数 描述
-f 忽略不存在的文件，不会出现警告信息
-i 删除前会询问用户是否操作
-r/R 递归删除
-v 显示指令的详细执行过程

删除前逐一询问确认

[root@localhost ~]# rm -i test.txt.bz2 
rm: remove regular file `test.txt.bz2'?
1
2

直接删除，不会有任何提示

[root@localhost ~]# rm -f test.txt.bz2
1

递归删除目录及目录下所有文件

[root@localhost ~]# mkdir /data/log
[root@localhost ~]# rm -rf /data/log
1
2

删除当前目录下所有文件

[root@localhost ~]# rm -rf *
1

清空系统中所有的文件（谨慎）

[root@localhost ~]# rm -rf /*
1

# rmdir

rmdir 命令作用是删除空的目录

注意：rmdir命令只能删除空目录。当要删除非空目录时，就要使用带有 -R 选项的rm命令。

rmdir命令的 -p 参数可以递归删除指定的多级目录，但是要求每个目录也必须是空目录。

语法格式 : rmdir [参数] [目录名称]

常用参数：

参数 描述
-p 用递归的方式删除指定的目录路径中的所有父级目录，非空则报错
-v 显示命令的详细执行过程

删除空目录

[root@localhost ~]# rmdir dir
1

递归删除指定的目录树

[root@localhost ~]# rmdir -p dir/dir_1/dir_2
1

显示指令详细执行过程：

[root@localhost ~]# rmdir -v dir
rmdir: 正在删除目录 'dir'
[root@localhost ~]# rmdir -p -v dir/dir_1/dir_2
rmdir: 正在删除目录 'dir/dir_1/dir_2
rmdir: 正在删除目录 'dir/dir_1'
rmdir: 正在删除目录 'dir_2'

# &、&&、|、||

& 表示任务在后台执行，如要在后台运行 redis-server 软件，则用 redis-server &

&& 表示前一条命令执行成功时，才执行后一条命令

| 表示管道，上一条命令的输出，作为下一条命令参数

|| 表示上一条命令执行失败后，才执行下一条命令

# chgrp

更改文件属组

chgrp 是英语单词 change group 的缩写，命令的作用和其中文释义一样，为用于变更文件或目录的所属群组。

语法格式：chgrp [参数] [目录]

常用参数：

参数 描述
-f 不显示错误信息
-R 递归处理，将指定目录下的所有文件及子目录一并处理

改变文件的群组属性

[root@localhost ~]# chgrp zyq file
1

将 /root/zyq 及其子目录下的所有文件的用户组改为zyq：

[root@linuxcool ~]# chgrp -R zyq /usr/zyq

# chown

Linux/Unix 属于多用户多任务操作系统，所有的文件皆有拥有者。利用 chown 命令可以将指定文件的拥有者改为指定的用户或组，用户可以是用户名或者用户ID，组可以是组名或者组ID，文件是以空格分开的要改变权限的文件列表，支持通配符。 一般来说，这个指令仅限系统管理者(root)所使用，普通用户没有权限改变文件所属者及所属组。

语法格式：chown [参数]

常用参数：

参数 描述
-R 对目前目录下的所有文件与子目录进行相同的拥有者变更
-f 若该文件拥有者无法被更改也不要显示错误讯息
-h 只对于连结(link)进行变更，而非该 link 真正指向的文件

将 test.txt 文件用户组与用户都改为 root

[root@localhost ~]# ll test.txt.bz2 
-rw-r--r-- 1 zyq zyq 56 Jul 22 20:17 test.txt.bz2
[root@localhost ~]# chown root:root test.txt.bz2    
[root@localhost ~]# ll test.txt.bz2
-rw-r--r-- 1 root root 56 Jul 22 20:17 test.txt.bz2
1
2
3
4
5

将当前目录下所有文件的拥有者都改为 zyq，用户组改为 lovezyq

[root@localhost ~]# chown -R zyq:lovezyq *
1

# chmod

chmod命令的英文原意是 change the permissions mode of a file 我们简称为 change mode

意为用来改变文件或目录权限的命令，但是只有文件的属主和超级用户root才能执行这个命令。有两种模式，一种是采用权限字母和操作符表达式；另一种是采用数字。

语法格式： chmod [参数] [文件]

常用参数：

参数 描述
-f 若该文件权限无法被更改也不显示错误讯息
-R 对目前目录下的所有文件与子目录进行相同的权限变更(即以递回的方式逐个变更)

a+r

属性分类

  • 文件所有者 u(user)
    • 文件所属组 g(group)
    • 其他人 o(other)
    • 所有的人 a(all)

操作权限

  • 读 r(read)
  • 写 w(write)
  • 执行 x(execute)

将档案 file1.txt 设为所有人皆可读取

[root@localhost ~]# chmod a+r file.txt
1

将目前目录下的所有文件与子目录皆设为任何人可读取

[root@localhost ~]# chmod -R a+r *
1

将 file.txt 设定为只有该文件拥有者可以执行

[root@localhost ~]# chmod u+x file.txt
1

Linux文件属性有两种设置方法，一种是数字，一种是符号。

Linux 文件的基本权限就有九个，分别是 owner/group/others(拥有者/组/其他) 三种身份各有自己的 read/write/execute 权限。

文件的权限字符为： -rwxrwxrwx ， 这九个权限是三个三个一组的！其中，我们可以使用数字来代表各个权限，各权限的分数对照表如下：

  • r 为 4
  • w 为 2
  • x 为 1

每种身份(owner/group/others)各自的三个权限(r/w/x)分数是需要累加的，例如当权限为： -rwxrwx--- 分数则是：

  • owner = rwx = 4+2+1 = 7
  • group = rwx = 4+2+1 = 7
  • others= --- = 0+0+0 = 0

所以等一下我们设定权限的变更时，该文件的权限数字就是 770。变更权限的指令 chmod 的语法是这样的

[root@localhost ~]# chmod 770 file.txt
1

# 压缩和解压

一般Linux系统使用的最多的就是 .tar.gz .tar.bz2 .tar .zip .rar

# tar

参数 描述
-z 使用gzip的方式进行压缩, 压缩包的后缀一般指定为 gz，xxx.tar.gz
-j 使用bzip2的方式进行压缩, 压缩包的后缀一般指定为 bz2，xxx.tar.bz2
-c 创建压缩包
-x 解压缩
-v 压缩的时候显示log信息
-f 指定压缩包的名字

举例说明

压缩文件格式为 xxx.tar.gz 的压缩包

tar -zcvf file.tar.gz file
1

压缩目录格式为 xxx.tar.gz 的压缩包

tar -zcvf file.tar.gz file/
1

在当前目录下解压文件目录格式为 xxx.tar.gz 的压缩包

tar -zxvf xxx.tar.gz
1

# zip

zip解压缩软件需要自行安装，压缩使用 zip 命令，解压使用 unzip 命令

Centos系统可以使用 yum install zip 安装

Ubuntu系统可以使用 sudo apt-get install zip

举例说明

压缩文件格式为 xxx.zip

zip xxx.zip
1

解压文件

unzip xxx.zip [可选参数，解压到指定目录]
1

# rar

rar解压缩软件需要自行安装

Centos系统可以使用 yum install rar 安装

Ubuntu系统可以使用 sudo apt-get install rar

举例说明

压缩文件格式为 xxx.rar，若压缩的是目录则需要添加参数 -r 参数

rar a xxx.rar file
1
rar a -r xxx.rar file/
1

解压文件（解压路径可不填，若不填则在当前目录下解压，可指定目录解压）

rar x xxx.rar [解压路径]
1

# 查找与检索

# find

根据文件的特征，查找磁盘内对应的文件

  • 根据文件名查找
  • 根据文件类型查找
  • 根据文件大小查找
  • 根据目录的层级查找

语法格式：find 搜索路径 参数 属性值

文件名查找

参数为 -name

  • * 匹配0个或多个
  • ? 匹配一个

查找根目录下所有文件后缀的是 .txt 文件

find / -name *.txt
1

查找根目录下所有文件后缀的是 zyq.txt 文件

find / -name abc.txt
1

文件类型查找

参数为 -type

Linux7种文件类型

  • 普通文件 f
  • 目录 d
  • 软连接 l
  • 字符设备 c
  • 块设备 b
  • 管道 p
  • 套接字 s

查找根目录下所有文件类型为目录的文件

find / -type d
1

# grep

根据搜索指定的关键字，搜索文件内容，得到对应的文件名

如果搜索的是目录需要 -r 参数

如果想知道关键字出现在文件的哪一行 -n

语法格式：grep [-r] "搜索的关键字" 搜索的目录/文件

grep -r "kexec" /root/
1
