# 文件上传漏洞

# 前端验证

文件上传前，前端JS判断文件后缀名，从而判断是否为图片，抓包修改即可绕过

# Content-type方式绕过

如果通过判断 Content-type 方式是否为图片，在抓包时修改数据包中的 Content-type 参数中的值，改成 image/jpeg、image/png等图片格式即可绕过

# 文件上传黑名单

服务端设置了文件上传的黑名单，不能上传 php 文件时，上传 .php3,.php4,.php5,.phtml php2 等能够解析的文件格式即可绕过

JSP的脚本有 jspx,jspf jspa jsw jsv jtml 解析格式

ASP的脚本有 asa,cer,aspx asax ascx ashx asmx 解析格式

# ::$DATA(Windows文件流绕过)

NTFS交换数据流(ADS)，ADS是NTFS磁盘格式的一个特性，在NTFS文件系统下，每个文件都可以存在多个数据流。通俗的理解，就是其它文件可以“寄宿”在某个文件身上，而在资源管理器中却只能看到宿主文件，找不到寄宿文件。

echo abcd >> a.txt:b.txt
1

将abcd写入 a.txt:b.txt 生成一个 a.txt

然后再试试 a.txt::$DATA ::$DATA默认不修改文件流的情况 生成一个正常的 a.txt

利用Windows特性，可在后缀名中加"::$DATA"绕过

# %00和00截断绕过

在 php5.3.4 以下，并且 php.ini 中 magic_quotes_gpc: off 可 %00 截断

%00 实际上和 00 截断是一模一样的原理，只不过 %00 是经过URL编码的，%00 解码后就是 0x00 截断的那个字符

1.php%00.jpg 传参之后，有些过滤都是直接匹配字符串，他强行匹配到了结尾是.jpg，然后允许上传，但是php的函数去执行的时候他读取到 0x00 认为结束了，那么这个文件就变成了 1.php

注意：POST方式提交save_path，问题在于POST方式我们用%00却失效了，这是因为POST传参并不会URL解码，所以需要我们该Hex改为00

# 条件竞争绕过

上传文件后，最后因为过滤或者因为其他原因被删除了，可以使用条件竞争，实际上是和unlink，是和删除文件的函数进行竞争

假如不断的上传发包，同时也不断的访问那个上传上去的文件的地址，函数执行都是要时间的，这边上传上去，且没有删除，那个时间可能很短，被访问到了，就可以执行PHP了

可以上传一个php然后访问后，由这个php去写一个马

<?php $a = '<?php @eval($_POST[\'a\'])?>';file_put_contents('1.php',$a)?>
