当进行多表操作时，可以理解为把多个表联系成一个表，然后对这一个大表进行操作

# 多表关联

# 一对一

居民和居民身份证的关系，一个居民只有一个身份证，一个身份证只属于一个居民

# 一对多

班级和班级学生的关系，一个学生只属于一个班级，一个班级有很多学生

# 多对多

老师和学生的关系，一个学生有很多老师，一个老师有很多学生

# 笛卡尔积

笛卡尔积在SQL中的实现方式既是交叉连接(Cross Join)。所有连接方式都会先生成临时笛卡尔积表，笛卡尔积是关系代数里的一个概念，表示两个表中的每一行数据任意组合

多表连接会得到所有可能出现的行，没有做两个表之间的关联条件，所有记录都符合

select * from stu,class;
1

一般都会添加条件使用

select * from stu,class where stu.class_id = class.id
1
select * from stu as s,class as c where s.class_id = c.id
1

stu.class_id 为 stu 表中的 class_id 字段，也可以起表别名

查询所有班级的所有学生

select c.id,count(*) from stu as s INNER JOIN class as c

# 左连接

LEFT JOIN 全称：左外连接，左连接，左边的表的记录将会全部显示出来，右表只会显示符合搜索条件的记录，右表无值的地方则为NULL

查询没有发表过文章的学生

SELECT s.sname FROM stu as s
LEFT JOIN article as a
ON s.id = a.stu_id
WHERE a.id IS NULL;
1

# 右连接

RIGHT JOIN 全称：右外连接，右连接，和左连接相反，右边的表的记录将会全部显示出来，左表只会显示符合条件搜索的记录，左表无值的地方则为NULL

查询所有学生对应的班级，没有班级则显示为无

SELECT s.sname,if(s.class_id,c.cname,'无') FROM class as c
RIGHT JOIN stu as s
ON c.id = s.class_id;

# 自连接

自连接就是本表再复制一个表，对两个表进行操作

使用子查询，查找与张三在一个班级的同学

SELECT * FROM stu WHERE class_id = (SELECT class_id FROM stu WHERE sname = '张三') AND sname != '张三';
1

使用自连接，查找与张三在一个班级的同学

SELECT s.sname FROM stu as s
INNER JOIN stu as s1
ON s.class_id = s1.class_id
WHERE s1.sname = '张三'
AND s.sname != '张三';

使用自连接，查找与张三同年出生的同学

SELECT s1.sname FROM stu as s
INNER JOIN stu as s1
ON YEAR(s.birthday) = YEAR(s1.birthday)
WHERE s.sname = '张三'
