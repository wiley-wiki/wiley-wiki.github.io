# 外键约束简介和要求

外键是一个表的字段被另一个表的字段引用，外键将两个表中的值进行限制

举例：学生表和班级表，学生表依赖班级表，通过外键约束使班级表和学生表关联起来，当班级表数据发生变化时也影响学生表

  • 主表与子表的存储引擎需要一致
  • 存储引擎是InnoDB
  • 主表和子表外键字段类型需要一致
  • 子表外键字段需要使用索引（数据库会自动添加索引）

# 外键管理

# 创建表外键

CREATE TABLE stu(
	id int(11) NOT NULL AUTO_INCREMENT PRIMARY KEY,
	sname varchar(30) NOT NULL,
	class_id int(11) DEFAULT NULL,
	CONSTRAINT stu_class
	FOREIGN KEY (class_id)
	REFERENCES class(id)
	ON DELETE CASCADE
	ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8

# 外键关联词

选项 说明
CONSTRAINT 给外键字段起别名
FOREIGN KEY 外键，子表和主表关联的字段
REFERENCES 子表关联主表的字段
ON DELETE 当主表删除操作时的处理动作
ON UPDATE 当主表更新操作时的处理动作

# 外键处理动作

# ON DELETE

选项 说明
ON DELETE CASCADE 删除主表数据时，子表外键字段数据也同时删除
ON DELETE SET NULL 删除主表数据时，子表外键字段数据设置为NULL（子表字段类型允许为NULL）
ON DELETE NO ACTION / ON DELETE RESTRICT 删除主表数据时，子表不做处理。但是主表被关联字段值没有被子表外键字段值关联才可以删除主表数据

# ON UDDATE

选项 说明
ON UPDATE CASCADE 更新主表数据时，子表外键字段数据也会同时更新
ON UPDATE SET NULL 更新主表数据时，子表外键字段数据设置为NULL（子表字段类型允许为NULL）
ON UPDATE NO ACTION / ON UPDATE RESTRICT 更新主表数据时，子表不做处理。但是主表被关联字段值没有被子表外键字段值关联才可以更新主表数据
