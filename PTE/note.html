# 笔记记录

# 真实IP查找

# Ping服务

使用多地ping服务，看IP地址是否是唯一，不唯一则可能是使用了CDN服务

https://ping.aizhan.com/

# nslookup

使用nslookup检测，如果返回域名解析对应多个IP地址可能使用了CDN服务

# 查询历史DNS记录

查看IP和域名绑定的历史记录

利用SecurityTrails平台查询，地址：https://securitytrails.com/

除了历史的DNS记录，即使是当前的记录也可能泄漏原始服务器IP。例如，MX记录是一种常见的查找IP的方式。如果网站在与web相同的服务器和IP上托管自己的邮件服务器，那么原始服务器IP将在MX记录中。

# 子域名

很多网站的主域名做了CDN服务，其他子域名没有做CDN服务，所以可以通过子域名去查找真实IP

# 网络空间引擎搜索

常见的有以前的钟馗之眼，shodan，fofa搜索。以fofa为例，只需输入：title:“网站的title关键字”或者body：“网站的body特征”就可以找出fofa收录的有这些关键字的ip域名，很多时候能获取网站的真实IP

# SSL证书

假如你在123.com上托管了一个服务，原始服务器IP是136.23.63.44。 而CloudFlare则会为你提供DDoS保护，Web应用程序防火墙和其他一些安全服务，以保护你的服务免受攻击。为此，你的Web服务器就必须支持SSL并具有证书，此时CloudFlare与你的服务器之间的通信，就像你和CloudFlare之间的通信一样，会被加密（即没有灵活的SSL存在）。这看起来很安全，但问题是，当你在端口443（https://136.23.63.44:443）上直接连接到IP时，SSL证书就会被暴露。

# 国外主机解析域名

如果CDN只做了国内的，国外没做的话，可通过国外主机对域名解析获取到真实的IP地址

# 网站漏洞查找

  • 1）目标敏感文件泄露，例如：phpinfo之类的探针、GitHub信息泄露等。
  • 2）XSS盲打，命令执行反弹shell，SSRF等。
  • 3）无论是用社工还是其他手段，拿到了目标网站管理员在CDN的账号，从而在从CDN的配置中找到网站的真实IP。

# 服务器邮件发送

在网站找回密码或者其他可以让服务器发送邮件的地方，给自己发送一份邮件，在邮箱里可以看到发送方服务器的IP地址

# F5 LTM解码法

当服务器使用F5 LTM做负载均衡时，通过对set-cookie关键字的解码真实ip也可被获取，例如：Set-Cookie: BIGipServerpool_8.29_8030=487098378.24095.0000，先把第一小节的十进制数即487098378取出来，然后将其转为十六进制数1d08880a，接着从后至前，以此取四位数出来，也就是0a.88.08.1d，最后依次把他们转为十进制数10.136.8.29，也就是最后的真实IP

# 注册表技巧

命令注册表查看远程桌面服务是否开启

REG QUERY "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server" /v fDenyTSConnections

fDenyTSConnections REG_DWORD 0x1

0x1则表示远程桌面是关闭状态，如果是0x0则表示远程桌面是开启状态

命令注册表查看远程桌面服务端口号

REG QUERY "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-tcp" /v PortNumber

PortNumber REG_DWORD 0xd3d

0xd3d这个就是端口号的16进制，转换成10进制即可

d3d十进制为3389

# SSR或者V2翻墙结合Burp抓包

有时候需要测试翻墙才可以访问的网站，或者隐藏自己的真实ip的情况下，需要用到多重代理，既在挂了国外代理的情况下使用burp。 此时就需要正确配置这两者，才可以正常使用。

首先：我们最常使用的代理ip的工具就是ssr或者v2，默认1080端口，且自动配置全局socks5代理，开启v2代理后，浏览器代理设置成127.0.0.1:1080 即可成功使用代理服务器的ip了。 那么我们需要在使用了代理的情况下，如何使用burp抓包进行测试呢？

流程：设置浏览器的代理为BurpSuite的proxy代理使其能够正常抓包，然后为BurpSuite再设置一个上游代理就可以，这样访问外网时，请求包会先通过Burpsuite再走向SSR，最后经过SSR服务器到达外网。 1.首先设置浏览器的代理为BurpSuite下proxy对应的ip跟端口，既127.0.0.1:8080 //首先保证burp能够正常抓包 2.设置burp内置的上游代理服务器： 在BurpSuite的User options下的Connections页面中，有个Upstream Proxy Servers配置项，点击Upstream Proxy Servers列表框顶部的Add按钮，打开Edit upstream proxy rule然后，这里一共有8个设置项，一般情况下只需关注前4个。 Destination host ：这里填入目标网站域名或IP，支持通配符，在本例中，我直接填写* Proxy host ：填入SSR或者V2代理服务器的IP地址，即127.0.0.1 Proxy port ：填入SSR或者v2的代理端口，即1080 //特别注意端口需要跟实际使用的端口一致 Authentication type ：这里选择认证类型，由于SSR本地代理无需认证，这是选择无 最后单击确定即可。

需要注意的地方，有些朋友按照这个方法配置后还是无法正常抓包，那是因为ssr或者v2配置的模式不对，代理模式应该选择直连模式，不能用全局模式(socks5全局)。需要把ssr或者v2工具的http代理模式改成：仅开启http代理，不自动配置代理服务器(直连模式)。
